## 2. T0 design decisions

1. **Representation: θ-form, not monic D-form.** The E-002 resolution said "rebuild on
   `DiffOp3`/`symSquare` content", but encoding the candidates in the monic-`D`
   `DiffOp3` type hits the RatFunc normalization trap (escalation trigger **E-04b** in
   `briefs/S1-04.md`). The ODE **order** — the only fact Theorem 1 consumes — is
   representation-independent and directly readable in θ-form. Decision: encode the
   operators as elements of ℚ[z][θ] (`Polynomial (Polynomial ℚ)`, outer `X` = θ), verbatim
   in the fetched literature shapes:
   - Cooper (order-3): `θ³ − z(2θ+1)(aθ²+aθ+b) + z²(c(θ+1)³+d(θ+1))` (Gorodetsky eq 1.7)
   - Zagier (order-2): `θ² − z(Aθ²+Aθ+λ) + Bz²(θ+1)²` (Gorodetsky eq 1.6)
   Both shapes were fetched from the primary source during S1-04 (recorded in
   `refs/cooper_sequences.md` and E-004) — not recalled. The S1-04 `symSquare`/`DiffOp2/3`
   API is untouched and remains the C3 vehicle; a θ↔D bridge, if ever needed, is its own
   T0 design item.
2. **No replacement axiom.** The order facts are *provable*: the θ-leading coefficient is
   `1 − 2a·z + c·z²` (Cooper) / `1 − A·z + B·z²` (Zagier), whose constant term is 1, hence
   nonzero for EVERY parameter choice. So `natDegree = 3` (resp. 2) holds generically —
   kernel theorems `cooperThetaOperator_natDegree` / `zagierThetaOperator_natDegree`, no
   empirical input consumed. The S₁,₂ parameters (11, 3, −1) become a sourced *definition*
   (`S12_zagier_params`, AutoEvolve pipeline provenance); the residual empirical claim
   ("the actual pipeline sequence is annihilated by this operator") is documented as
   Tier B in the docstring, not smuggled into an axiom.
3. **Honest boundary, stated everywhere it matters:** minimality of each operator for its
   sequence, and the elliptic/K3 period identification, are NOT established — Tier B,
   S1-05 bridge scope. Theorem-1 docstrings now say "order classification of the encoded
   operators", nothing more.
